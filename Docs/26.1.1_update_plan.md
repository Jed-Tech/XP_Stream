# Minecraft 26.1.1 Update Plan (XP_Stream)

Plan for updating the workspace to the `26.1.1` line, moving **NeoForge** to `26.1.1.1-beta`, validating **Fabric** on the same family, and releasing **XP_Stream 1.1.1**.

**Status:** Planned  
**Scope:** `xp_stream` only for the mod version bump; repo-wide loader/tooling verification for Fabric + NeoForge

---

## Goals

- Keep the repo aligned with the Minecraft `26.1.1` family.
- Update **NeoForge** from `26.1.0.5-beta` to `26.1.1.1-beta`.
- Re-verify **Fabric** against the same `26.1.1` update line.
- Bump `mods/xp_stream/gradle.properties` from `mod_version=1.1.0` to `mod_version=1.1.1`.
- Record release notes and compatibility after testing.

---

## Current -> target

| Item | Current | Target |
|------|---------|--------|
| `minecraft_version` | `26.1` | `26.1.1` if resolvable for Fabric tooling; otherwise use the exact 26.1.1-compatible identifier that resolves in Loom |
| `neoforge_version` | `26.1.0.5-beta` | `26.1.1.1-beta` |
| `mods/xp_stream/gradle.properties` `mod_version` | `1.1.0` | `1.1.1` |

| NeoForge verification | Built for `26.1.0.5-beta` | Rebuild and smoke-test on `26.1.1.1-beta` |

## Planned changes

### 1. Root version pins

Update repo-wide properties in `gradle.properties`:

- Set `neoforge_version=26.1.1.1-beta`
- Move `minecraft_version` to the matching `26.1.1` family value used for local dev/builds
- Keep Java 25
- Leave Fabric properties unchanged unless the `26.1.1` update requires newer Fabric loader/Loom/API values during build verification

### 2. XP_Stream release version

Update `mods/xp_stream/gradle.properties`:

- Change `mod_version=1.1.0` -> `mod_version=1.1.1`

Expected effect:

- Fabric jar version becomes `1.1.1`
- NeoForge jar version becomes `1.1.1`
- Expanded loader metadata follows the same mod version

### 3. Metadata sanity check

Confirm existing broad compatibility policy still holds:

- `mods/xp_stream/fabric/src/main/resources/fabric.mod.json`
- `mods/xp_stream/neoforge/src/main/resources/META-INF/neoforge.mods.toml`
- `Docs/broad_minecraft_family_policy.md`

Checks:

- Fabric dependency ranges still make sense for the `26.1.1` line
- NeoForge minimum version expands correctly from `${neoforge_version}`
- No loader metadata needs tightening beyond the compile pin

### 4. Changelog

Add a new top entry to `CHANGELOG.md` for `1.1.1` that captures:

- Minecraft `26.1.1` update support
- NeoForge update to `26.1.1.1-beta`
- Fabric compatibility/test status
- Any user-visible behavior changes if they appear during verification

## Verification plan

Use `just` commands from the repo root.

### Build checks

1. `just build-all`
2. If needed, target XP_Stream loader runs individually:
   - `just xp-stream-fabric-client`
   - `just xp-stream-neoforge-client`

### What to verify

- Fabric build completes successfully
- NeoForge build completes successfully with `26.1.1.1-beta`
- XP orb burst pickup behavior still works
- No new mixin/config/runtime startup errors
- Output artifacts reflect `1.1.1`

Artifact expectation examples:

- `mods/xp_stream/fabric/build/libs/...1.1.1.jar`
- `mods/xp_stream/neoforge/build/libs/...1.1.1.jar`

---

## Risks / watch items

- Fabric may still resolve against a slightly different version string than plain `26.1.1`; use the resolvable value required by Loom if necessary
- NeoForge patch updates can require a matching `minecraft_version` change in root `gradle.properties`
- If Fabric or NeoForge updated mappings/tooling surface API drift, code changes may be needed even if this starts as a version-only bump
