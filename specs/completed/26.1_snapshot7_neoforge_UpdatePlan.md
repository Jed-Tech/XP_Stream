# NeoForge 26.1 Snapshot 7 Update Plan

## Overview

This document outlines the plan to align XP_Stream’s **NeoForge** build with **NeoForge 26.1.0.0-alpha.12+snapshot-7**, which bundles **Minecraft 26.1 Snapshot 7**. NeoForge ships one version per Minecraft snapshot; supporting “their bundle” means building and testing against this exact pair.

**Target NeoForge version:** `26.1.0.0-alpha.12+snapshot-7`  
**Target Minecraft version (for NeoForge bundle):** `26.1-snapshot-7`  
**Previous NeoForge version:** `26.1.0.0-alpha.5+snapshot-2`  
**Java requirement:** Java 25 (already configured)

---

## Why “Support Their Bundle”

- Each NeoForge 26.1 build is tied to one Minecraft snapshot (e.g. `alpha.12+snapshot-7` → 26.1-snapshot-7).
- Building the mod against the **same** Minecraft version that NeoForge bundles avoids version mismatches and ensures the mod runs on the correct runtime.
- Check NeoForge Maven `maven-metadata.xml` for `<latest>` / `<release>` to pick the current version: [NeoForge Maven](https://maven.neoforged.net/releases/net/neoforged/neoforge/).

---

## Compatibility Assessment

Based on review of **26.1 Snapshots 4–8** changelogs:

- **No changes** to `ExperienceOrb`, `playerTouch`, player pickup delay, or entity collision APIs that affect XP_Stream.
- Mod uses: `ExperienceOrb`, `playerTouch`, `ServerLevel`, `Player`, `AABB`, `getEntitiesOfClass`, and the player’s pickup-delay field.

**Conclusion:** No code changes expected. Version/config-only upgrade for the NeoForge side.
